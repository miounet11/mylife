# 相关参考域地图 V2

## 目标

把“对命理知识引擎有直接增益的相邻领域”排出主次，避免把资料库做成一锅粥。

当前建议分成三层：

1. 主增强域：心理学、哲学、历史学
2. 引擎增强域：AI、统计学、程序 / 软件工程
3. 扩展辅助域：星座学、医学

法律保留为低优先级储备域，不再占用主建设资源。

## 一、为什么优先级要改

如果目标是把命理站做成全球顶级知识库，关键不是“抓更多网页”，而是让整个系统在以下方面明显变强：

- 概念更清楚
- 问题理解更贴近用户真实表达
- 解释链更稳
- 引擎验证更可校准
- 工程化资料采集和归一能力更强

从这个角度看，最有价值的并不是法律，而是下面这些域。

## 二、主增强域

### 心理学

价值：

- 提升情绪、关系、压力、自我认知相关问题的理解质量
- 建立安全表达边界，减少把复杂心理问题神秘化
- 为“天时地利人和”中的“人和”提供更稳的解释结构

接入方式：

- 术语归一层
- 情绪与关系问题地图
- 风险提示层
- 用户表达清洗层

不该做的事：

- 不把心理学话术伪装成命理结论
- 不给诊疗建议

### 哲学

价值：

- 强化概念定义、认识论边界、解释方法
- 帮助区分“经验判断”“象数解释”“模型假设”“价值判断”
- 让整个知识库不只是会收集，还会讲清楚

接入方式：

- 概念骨架层
- 方法论层
- 世界观与问题框架层

### 历史学

价值：

- 为经典、流派、人物、术语建立时间线和历史语境
- 帮助区分“原始语境”和“后世解释”
- 支撑书目史、思想史、流派演化图谱

接入方式：

- 史料索引层
- 时间线层
- 思想与流派演化层

## 三、引擎增强域

### AI

价值：

- 直接服务资料引擎、抽取链、Agent、RAG、评估系统
- 支撑多源融合、引用追踪、问题聚类、知识链生成

接入方式：

- 参考智能层
- 抽取与聚类层
- 检索增强和评测层

### 统计学

价值：

- 建立评分、置信度、校准和验证闭环
- 让“准确性、有效性、权威性”不只停留在口号

接入方式：

- 状态向量校准
- 证据强度分级
- 问题热度和来源可靠性评分

### 程序 / 软件工程

价值：

- 直接支撑抓取、清洗、去重、归档、检索、发布
- 决定知识库能否稳定扩张，而不是一波流采集

接入方式：

- 采集器实现层
- 数据清洗与任务编排层
- 资料库基础设施层

## 四、扩展辅助域

### 星座学

价值：

- 用户兴趣入口强
- 适合做跨体系对照和书单扩展

定位：

- 兴趣入口层
- 术语映射层
- 问题发现层

### 医学

价值：

- 用于健康话题安全边界和术语校正
- 帮助处理睡眠、压力、恢复、风险提示这些表达

定位：

- 安全边界层
- 风险提示层
- 术语规范层

不该做的事：

- 不做诊断
- 不做治疗建议

## 五、当前优先级

建议主线排序如下：

1. `玄学 / 命理 / 易学`
2. `心理学`
3. `哲学`
4. `历史学`
5. `AI`
6. `统计学`
7. `程序 / 软件工程`
8. `星座学 / 西方占星`
9. `医学 / 健康参考`
10. `法律 / 合规参考`

## 六、进入产品的方式

### 直接进入主知识引擎

- 命理 / 易学 / 玄学
- 心理学
- 哲学
- 历史学

### 直接进入引擎能力层

- AI
- 统计学
- 程序 / 软件工程

### 进入补充解释和边界层

- 星座学
- 医学

### 暂时只做储备

- 法律

## 七、最终判断

当前最值得立刻重兵推进的，不是更多平台抓取，而是三条建设线并进：

- 命理本体经典、书目、问题、流派的主骨架
- 心理学 / 哲学 / 历史学的解释与概念支撑层
- AI / 统计 / 工程的资料引擎能力层

这样做出来的不是“内容站复制品”，而是一个能持续吸收、校正、解释、验证的知识系统。
