# Release Notes: v2.0.0

发布日期：2026-03-13

## 版本定位

`v2.0.0` 不是一次局部修补，而是产品主链的一次正式升级。

这一版同时完成了两件事：

1. 把面向用户的核心体验拉到统一、完整、可闭环的产品状态。
2. 把面向团队的行为数据、验证数据和经营观察能力真正接进系统。

从这一版开始，我们不再只看“功能是否能用”，而是开始持续看：

- 用户从哪里进入
- 用户在哪一步流失
- 用户是否继续深问
- 用户是否把报告转成事件
- 用户是否回填验证结果
- 哪些报告版本、推理层和交互路径效果更好

## 本版核心升级

### 1. 统一产品主链

已经统一并加强以下核心页面与流程：

- 首页
- 分析页
- 结果页
- 聊天页
- 事件页
- 登录 / 邮箱验证码流程

目标不是“页面都存在”，而是让用户从进入产品到生成报告、继续追问、沉淀事件、回填验证形成完整闭环。

### 2. 报告系统升级为 V2 主链

报告系统已从单层解释升级为多层组合：

- 结构化命理引擎
- 人生 K 线引擎
- 天时地利人和上下文
- deterministic 专家层
- 并发 Agent 能力
- review / repair / verify 闭环
- 版本化报告元数据

结果页现在会显式记录并展示：

- 报告版本
- 命理引擎版本
- K 线版本
- 文本增强状态
- 推理层状态
- 一致性评分
- 升级重算来源

### 3. 人生 K 线升级为主轴

这一版不再把人生 K 线当成辅助图表。

已经把以下内容向 K 线主轴靠拢：

- 关键锚点年份
- 优先窗口
- 高低谷阶段
- 当前大运与流年互动
- 建议动作节奏
- 报告正文叙事顺序

### 4. 天时地利人和正式进系统

这一版把“天时地利人和”从概念文案变成系统参数，已纳入：

- 节气
- 立春边界
- 流年
- 国运 / 宏观周期
- 行业周期
- 地理环境
- 气候偏性
- 有利方位
- 人生阶段

这些参数已经开始进入：

- 结果页正文
- 专家层解释
- deterministic fallback
- 审查与校验逻辑

### 5. 聊天系统升级

聊天部分已升级为更成熟的对话产品逻辑，支持：

- 默认滚动到底部
- Markdown / GFM 渲染增强
- 编辑已发送消息
- 删除消息
- 重生成回答
- 继续围绕报告上下文深问
- 将聊天结论沉淀为事件

### 6. 行为分析与经营后台正式上线

从 `v2.0.0` 开始，系统开始持续记录并聚合关键行为：

- 首页访问
- 分析页访问
- 聊天页访问
- 事件页访问
- 提交测算
- 生成报告
- 浏览结果页
- 点击结果页 CTA
- 升级重算
- 发送聊天消息
- 聊天编辑 / 重生成 / 删除
- 结果页转事件
- 聊天转事件
- 回填验证结果
- 邮箱验证码请求与验证
- 订阅邮件

后台新增可直接观察的经营维度：

- 核心漏斗
- 页面访问结构
- 分析入口偏好
- 推理层覆盖
- 聊天动作结构
- 结果页 CTA 表现
- 事件来源验证拆解
- 偏差原因分布
- 待验证回收队列
- 报告版本结构

## 本版最重要的产品价值

### 对用户

- 更快进入分析
- 更少干扰填写
- 更清楚理解报告
- 更容易从结果页进入聊天和事件闭环
- 更容易回看、验证和纠偏

### 对团队

- 开始真正看到用户行为而不是凭感觉判断
- 开始积累验证样本而不是只看“报告生成量”
- 开始比较不同版本、不同推理层、不同入口路径的效果
- 开始具备持续迭代的经营数据基础

## 这一版开始重点观察的指标

### 入口层

- 首页访问量
- 分析页访问量
- 分析页访问到提交测算的转化
- 真太阳时选项使用率
- 子时分日选项使用率

### 报告层

- 报告生成量
- 报告浏览量
- 报告浏览到聊天的转化
- 报告浏览到事件沉淀的转化
- 升级重算触发量
- 各报告版本占比
- 各推理层占比

### 聊天层

- 直接提问量
- 编辑重提比例
- 重生成比例
- 删除比例
- 聊天到事件沉淀的转化

### 闭环层

- 事件总量
- 报告关联事件量
- 聊天来源事件量
- 验证准确量
- 验证偏差量
- 待验证量
- 偏差原因结构

## 下一轮迭代建议

基于 `v2.0.0` 之后开始沉淀的数据，下一轮优先关注：

1. 结果页到聊天、事件的真实转化是否偏低。
2. 哪些推理层组合带来的用户后续动作更多。
3. 哪些偏差原因最集中，是否对应引擎、上下文或表达问题。
4. 哪类入口配置更容易得到高质量报告与更高闭环率。
5. 聊天里的编辑 / 重生成是否说明答案质量或表达方式仍有系统性问题。
