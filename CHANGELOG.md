# Changelog

## v2.0.1 - 2026-03-14

### Added
- 新增当前版项目文档：
  - `README.md`
  - `docs/ARCHITECTURE.md`
  - `docs/DEVELOPMENT.md`
  - `docs/OPERATIONS.md`

### Changed
- 统一当前版本的项目说明、开发说明、运维说明与文档入口。

### Removed
- 删除一批已失效的旧阶段文档、旧迁移文档、旧进度报告与旧文案草稿，避免继续误导开发与运维。

## v2.0.0 - 2026-03-13

### Added
- 统一首页、分析页、结果页、聊天页、事件页的产品体验与主站视觉。
- 上线 `agentic report` 体系，支持 deterministic 专家层、并发 Agent、审查修复与一致性校验。
- 上线报告版本化能力，支持 `reportVersion`、引擎构建信息、升级重算。
- 上线结果页报告增强模块，包括推理层状态、天地人上下文、人生 K 线主轴和事件沉淀入口。
- 上线聊天增强能力，包括消息编辑、删除、重生成、Markdown/GFM 渲染、上下文加载与事件沉淀。
- 上线经营后台 `/admin/analytics`，开始统一观察分析、聊天、事件、验证闭环和版本结构。
- 上线前端/后端埋点主链，开始记录页面访问、分析提交、报告生成、报告浏览、聊天动作、事件沉淀、验证反馈、订阅与登录验证行为。
- 补充 V2 / V3 规划与执行文档，固定产品升级路线。

### Changed
- 分析入口改为更贴近目标站点的低干扰快速填写流。
- 结果页不再把 fallback 报告错误表达为“仅基础引擎”，改为 `基础引擎 / Deterministic 专家层 / 并发 Agent` 三态。
- 人生 K 线从附属展示升级为报告主轴，并将时间窗口、节气、方位、宏观周期回灌到正文。
- 报告文本从“基础引擎解释”升级为“命理引擎 + 上下文补强 + 专家层整合”。
- 聊天交互改为更接近现代对话产品的消息流逻辑。

### Improved
- 强化 deterministic review / verify 规则，增加对锚点年份、优先窗口、行业周期、流年、地理环境的显式约束。
- 改善结果页内容密度、解释清晰度与用户后续动作入口。
- 改善行为分析维度，从单纯事件计数扩展到页面访问、漏斗、推理层覆盖、聊天动作结构和 CTA 表现。

### Notes
- 本版本是产品大版本升级节点，也是后续数据驱动迭代的基准版本。
- 后续优化将重点依赖该版本开始沉淀的行为数据、验证数据和偏差样本。
